# Load data in background worker using Polars/PyArrow for responsive viewing

## Task

Load table data (from memory or disk) in a Textual worker so the UI stays responsive, especially for large tables.

## Context

Spike highlights Textual Workers (`run_worker` / `@work`) for file IO/compute. Using Polars provides fast CSV parsing with automatic type inference and lazy evaluation via `.head(N)`.

For in-memory tables (Polars DataFrame), we can directly sample rows without IO.

## Acceptance Criteria

- [ ] Data loading runs via `run_worker(..., exclusive=True)` or `@work`
- [ ] For disk sources: Use `polars.read_csv(source, n_rows=N)` or `pl.scan_csv().head(N).collect()`
- [ ] For in-memory sources: Use existing DataFrame's `.head(N)`
- [ ] Convert Polars/PyArrow data to Python lists for Textual DataTable consumption
- [ ] UI shows a loading/progress indicator while worker runs
- [ ] Worker errors are handled without crashing the app
- [ ] Row cap is enforced (configurable, default 200 rows)

## Implementation Notes

```python
# Disk source (CSV)
df = pl.read_csv(path, n_rows=limit)
# OR lazy (for very large files)
df = pl.scan_csv(path).head(limit).collect()

# In-memory source (existing DataFrame)
df = existing_df.head(limit)

# Convert to rows for DataTable
rows = df.to_numpy().tolist()
columns = df.columns
```
