# Implement Polars query executor (in-memory + scan_csv fallback)

## Task

Implement a query executor that applies the query model to:
- in-memory Polars DataFrames, and
- disk-backed CSV tables via `pl.scan_csv()` / `pl.read_csv()`.

## Context

Polars is preferred for MVP. Use lazy scans where possible and enforce row limits.

## Acceptance Criteria

- [ ] Executor can load data from a TableProvider (memory or disk)
- [ ] Applies filters (`==`, `contains`, `>`, `>=`, `<`, `<=`)
- [ ] Applies sort (single column)
- [ ] Applies groupby + basic aggregations (count, sum, mean)
- [ ] Uses lazy operations where possible and only collects `head(N)`
- [ ] Returns a Polars DataFrame suitable for DataTable display

## Constraints

- Avoid full table collects; always cap output rows
